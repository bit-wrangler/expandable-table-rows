Let's say you'd like to create a report with a data table that allows a variable amount of depth to the portrayed information. As an example, if we only have a single variable of interest, like daily temperature, we could take a whole month's worth of data and summarize it by taking the mean value. This lets us reduce the number of rows at the cost of granularity in the data. What if we wanted to display data in summarized form, but also wanted to give an option of viewing the detailed data set?

That was the motivation behind this exercise. There were two things I wanted to consider here:

  • How the data should be served by server
  • How the data should be processed on the cliend
